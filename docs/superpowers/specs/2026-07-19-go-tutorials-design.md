## 1. Goal

Scaffold a production-ready Go RESTful service using **Gin** (HTTP) + **GORM** (Postgres via pgx),
organized as a **feature-based modular** codebase so multiple sub-systems (phân hệ) can be added
without restructuring. The layout follows senior Go conventions: one-directional dependency flow,
`internal/` for protected business code, `pkg/` for reusable libraries.

Out of scope for this iteration: comprehensive test suite, CI/CD pipeline (deferred).

### Dependency rule

`handler → service → repository → db`. Handlers never call GORM directly.
Each feature owns its `model/dto/handler/service/repository/routes` so it can be cut into a
module/microservice later. All errors funnel through `common/errors`; all responses through
`common/response` envelope.

## 5. Data Flow, Errors, API Contract

### Request pipeline
`requestID → cors → logger → recover → ratelimit → (auth if protected) → handler → service → repository/cache`

### Error model (`internal/common/errors`)
```go
type AppError struct {
    Code    int    // internal code (1001 user_exists, 1002 invalid_credentials, 2001 not_found...)
    Message string // user-facing message
    HTTP    int    // http status
    Cause   error  // original, logged internally
}
```
Auto mapping: invalid credentials/forbidden/unauthorized → 401/403, not found → 404, conflict → 409,
validation → 400, else → 500.

### Response envelope (all endpoints)
```json
{ "code": 0, "message": "ok", "data": { } }
{ "code": 1001, "message": "email already exists", "data": null }
```

### API versioning
Prefix `/api/v1/...`.

### Auth endpoints (JWT + Refresh)
- `POST /api/v1/auth/register` — `{email, password, name, role?}` → user + access + refresh
- `POST /api/v1/auth/login` — `{email, password}` → `{access_token, refresh_token, expires_in}`
- `POST /api/v1/auth/refresh` — `{refresh_token}` → new token pair (rotation; old refresh blacklisted in Redis TTL=expires_in)
- `POST /api/v1/auth/logout` — revoke refresh token
- `POST /api/v1/auth/forgot-password` — generate UUID token (TTL 15m) in Redis; reset link logged to console
- `POST /api/v1/auth/reset-password` — `{token, new_password}` → update hash, revoke all user refresh tokens

### User endpoints
- `POST /api/v1/users` — create
- `GET /api/v1/users` — list (admin, paging + filter)
- `GET /api/v1/users/:id` — detail
- `PUT /api/v1/users/:id` — update
- `DELETE /api/v1/users/:id` — soft delete (GORM)
- `GET /api/v1/users/me` — current user from JWT

### RBAC
- Models `Role` (admin/user) + `Permission` + `role_permissions`.
- Middleware `RequireRole("admin")` checks JWT claims.
- `register` accepts optional `role` (default `user`).

### Audit log
- Middleware `audit.go` writes `{user_id, action, resource, method, path, status, latency_ms, ip, request_id}`
  to `audit_logs` (GORM). Toggle via `AUDIT_ENABLED=true`; skips `/healthz`, `/metrics`.

### File upload (local-only)
- `POST /api/v1/uploads` — multipart/form-data → `{url, size, mime, width?, height?}`
- `GET /static/*` — serve via `gin.Static`
- `Storage` interface in `common/storage`; `local.go` impl; UUID name, `YYYY/MM/<uuid>.<ext>` path,
  MIME whitelist (jpeg/png/webp/pdf), default max 10MB (configurable), real MIME via magic number.

### Paging & filter
- Query: `?page=1&page_size=20&sort=-created_at&q=keyword`
- Response data: `{items:[...], total, page, page_size}`

### Health & ops
- `GET /healthz`, `GET /readyz` (DB+Redis ping), `GET /metrics` (Prometheus), `GET /swagger/*any`

## 8. Completion Criteria

- `go build ./...` and `go mod tidy` clean.
- `make migrate-up` creates schema in local Postgres.
- `make run` starts server; `GET /healthz` returns 200; `GET /swagger/index.html` opens.
- `POST /api/v1/auth/register` → `login` → `GET /api/v1/users/me` returns user via JWT.
- `POST /api/v1/uploads` uploads a file locally.
